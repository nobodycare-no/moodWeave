# MoodWeave 下一轮迭代方案

> 创建日期：2026-05-30  
> 建议接手方：SOLO  
> 迭代定位：从“可演示的本地情绪板”推进到“可长期使用的创作工作流”

## 判断

当前项目已经具备完整 MVP：多 Board、图片/文本卡片、素材库、逻辑连线、AI 生图、PNG 导出、本地持久化和一键启动。下一轮不建议继续零散堆按钮，而应优先补齐高频创作工具最影响信任感的部分：

- 数据能备份、迁移、恢复。
- 误操作能撤销。
- 卡片编辑更高效。
- AI 生成结果可追溯、可再生成。
- 导入导出与回归验证形成稳定交接标准。

## 目标

让用户可以放心把 MoodWeave 当作真实灵感工作台使用，而不是一次性 Demo。下一轮完成后，应能在不同浏览器或电脑间迁移项目数据，能撤回常见误操作，能追踪 AI 图片来自什么 prompt，并能用一组固定回归场景验证核心链路。

## 迭代步骤（8 步以内）

1. **数据备份与恢复**  
   增加完整备份导出/导入能力，覆盖 Boards、Assets、Connections、IndexedDB 中的 `mw-image://` 图片引用。建议导出为单个 JSON 包或 ZIP 包；导入时先做结构校验、版本判断和错误提示，再写入本地存储。

2. **存储模型版本化**  
   在本地数据中引入 `schemaVersion`，统一 Board / Asset / Image metadata 的 normalize 与 migration 入口。后续新增字段时必须通过迁移函数兼容旧数据，避免每个 composable 各自补丁式兼容。

3. **Undo / Redo 操作历史**  
   为当前 Board 建立轻量历史栈，优先覆盖添加、删除、移动、文本编辑、连接创建、连接标签编辑。接入 `Ctrl/Cmd+Z`、`Ctrl/Cmd+Shift+Z` 或 `Ctrl/Cmd+Y`，并在工具栏提供可见入口。

4. **画布编辑效率提升**  
   增加复制卡片、键盘删除、Esc 取消连接/选择、置顶/置底、卡片尺寸调整。新增卡片和素材插入应优先落在当前视口中心附近，减少用户反复拖拽寻找内容。

5. **素材库管理增强**  
   支持素材重命名、搜索/筛选、删除确认和基础标签。素材库不只保存内容，还应保存来源信息，例如来自哪个 Board、哪次 AI 生成、或哪张卡片。

6. **AI 生成可追溯与再生成**  
   为 AI 生成的图片卡片/素材记录 prompt、model、size、quality、createdAt、sourceTextCardId 等元数据。选中图片或关联文本时，支持从原 prompt 再生成变体；后续可把连接线标签组合进 prompt。

7. **导出与导入体验打磨**  
   PNG 导出增加背景模式、倍率和导出范围选项，例如整板/选中内容。备份导入成功后给出摘要：导入了多少 Boards、Assets、Images，是否跳过重复项或坏数据。

8. **回归验证基线**  
   建立固定验收脚本或文档化 smoke test：启动、创建 Board、添加图片/文本、连接并标注、AI 缺 key 提示、导出 PNG、备份导出、备份导入、刷新后数据仍在。每次交付至少跑 `npm run build` 和这组浏览器实测。

## 非目标

- 暂不做用户登录、云同步、多人协作。
- 暂不做完整移动端重设计。
- 暂不引入大型状态管理库，除非 Undo / Redo 和备份导入已经让现有 composable 明显失控。
- 暂不把 AI key 放到生产后端；当前仍按本地私人使用模式处理。

## 推荐切入点

第一步建议从“数据备份与恢复 + 存储模型版本化”开始。原因是它直接解决本地数据不可迁移的问题，也会为后续 Undo / Redo、AI metadata、素材库增强打好统一数据边界。不要先做更花的 AI 玩法，否则数据结构会继续分散，后面迁移成本更高。

## 验收标准

- 用户可以导出当前全部本地数据，并在另一个空浏览器来源中导入后看到相同 Boards、卡片、素材、连线和本地图片。
- 常见误操作可撤销和重做，不破坏刷新后的持久化。
- AI 生成图片能查看原始 prompt/config，并能基于原记录再次生成。
- `npm run build` 通过。
- 浏览器 smoke test 覆盖核心创作链路，控制台无新增运行时错误。
